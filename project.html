Project Planning

서비스 기획과 사용자 흐름

ONWAVE는 대화가 업무로 이어지지 못하는 문제를 해결하기 위해 팀 커뮤니케이션의 구조를 다시 설계한 서비스입니다. 기획 페이지에서는 문제, 대상 사용자, 기능 구조, 이용 흐름을 중심으로 정리했습니다.

Background

문제 정의

Problem 01

대화가 여러 채널에 흩어짐

메신저, 회의록, 업무 관리 도구가 분리되면서 중요한 맥락을 다시 찾아야 하는 시간이 늘어납니다.

Problem 02

결정 사항이 메시지 속에 묻힘

합의된 내용이 별도로 정리되지 않으면 담당자와 마감일이 불명확해져 실행 속도가 느려집니다.

Problem 03

참여 상태를 파악하기 어려움

누가 의견을 냈는지, 어떤 이슈가 답변을 기다리는지 한눈에 확인하기 어렵습니다.

Target User

목표 사용자

프로젝트 매니저

팀 대화에서 결정 사항을 빠르게 정리하고 다음 액션을 지정해야 합니다.

기획자와 디자이너

아이디어, 피드백, 파일 공유가 흩어지지 않도록 한 흐름으로 관리합니다.

외부 협업자

필요한 채널에만 접근해 대화 맥락과 결정 결과를 확인합니다.

Core Strategy

서비스 해결 방향

01

대화 수집

프로젝트별 채널에서 메시지, 파일, 반응을 함께 기록합니다.

02

맥락 정리

질문과 결정 후보를 자동으로 묶어 중요한 흐름을 먼저 보여줍니다.

03

결정 저장

투표와 승인 결과를 결정 카드로 남겨 팀의 합의 근거를 보관합니다.

04

실행 연결

결정 내용을 담당자, 일정, 체크리스트가 있는 업무로 전환합니다.

Development Concept

웹 서비스 제작 방향

첨부한 웹 개발 이미지처럼 밝고 직관적인 기술 서비스 인상을 주기 위해 넓은 여백, 단순한 도형, 카드형 설명, 선명한 버튼을 사용했습니다.

  • 첫 화면에서 서비스 성격을 바로 이해
  • 기능 설명은 짧은 카드 단위로 정리
  • 이미지와 텍스트 비중을 균형 있게 배치

Feature

핵심 기능

#

주제 채널

팀, 프로젝트, 고객 이슈별로 대화 공간을 구분합니다.

?

질문 큐

답변이 필요한 메시지를 상단에 모아 놓치지 않게 합니다.

%

빠른 투표

찬반, 선호안, 일정 조율을 대화 안에서 바로 완료합니다.

+

결정 카드

합의된 내용을 업무와 연결해 다음 행동을 명확히 합니다.

User Journey

사용자 이용 흐름

Step 1

채널 입장

프로젝트별 대화방에서 이슈와 자료를 확인합니다.

Step 2

의견 수집

질문 큐와 반응 기능으로 팀원의 생각을 빠르게 모읍니다.

Step 3

결정 기록

투표 결과와 합의 내용을 결정 카드로 저장합니다.

Step 4

업무 실행

담당자와 일정이 연결된 실행 항목으로 전환합니다.

Planning Images

기획에 사용한 대표 이미지

서비스 메시지

서비스의 핵심 문장과 주요 기능을 한눈에 보여주는 메인 이미지입니다.

협업 흐름

대화, 결정, 실행이 이어지는 화면 구조를 보여주는 대시보드 이미지입니다.
